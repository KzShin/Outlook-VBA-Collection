# Outlook Template Mail Creator (modTemplateMail)
これは、OutlookのVBAマクロです。所定のフォルダに保存されたテンプレート（`.oft` または `.msg`）を読み込み、専用の選択ウィンドウから目的のテンプレートを選んで新規メールを作成します。

[![Image from Gyazo](https://i.gyazo.com/e0c8a76017d28db1f7733471ef1bd7c3.png)](https://i.gyazo.com/e0c8a76017d28db1f7733471ef1bd7c3.png)

作成時にテンプレート内の特定の変数（プレースホルダー）を**現在の日付や曜日、和暦（令和）に自動置換**し、さらに**強制的にテキスト形式のメールへ変換**して表示するため、日々の定型報告や案内メールの作成時間を大幅に短縮できます。

[![Image from Gyazo](https://i.gyazo.com/ee867e923be276d19890705873c1e6ac.png)](https://i.gyazo.com/ee867e923be276d19890705873c1e6ac.png)

## 特徴
* **専用選択UI**: テンプレート用フォルダ内のファイルを自動でリストアップし、見やすいウィンドウ（ユーザーフォーム）から選択できます。
* **日付の自動置換**: 件名や本文に含まれる `{yyyy}` などの専用変数を、実行時の日付に自動で置き換えます。
* **和暦・曜日の完全対応**: 環境設定に依存しない独自のロジックで、正しい和暦（`{ggge}` → 令和〇）や曜日（`{aaa}` → 月、火...）を出力します。
* **テキスト形式への強制変換**: テンプレートファイル自体がHTML形式やリッチテキスト形式で保存されていても、作成時に確実にプレーンテキスト（テキスト形式）へ変換します。
* **フォルダの自動生成**: 初回実行時に、自動的に設定されたテンプレート格納用フォルダを作成します。
* **共通ログ機能**: 共通モジュール `modLogger` を使用し、実行ログを記録します。
## 使えるプレースホルダー（変数）一覧
テンプレートの件名または本文に以下の文字列を入れておくと、マクロ実行時に自動置換されます。
※月や日は、要望に合わせて **0埋めなし（1桁の場合は1桁のまま）** で出力されます。
| 変数 | 出力例 (2026年3月2日 月曜日の場合) | 備考 |
| --- | --- | --- |
| `{yyyy}` | 2026 | 西暦（4桁） |
| `{yy}` | 26 | 西暦（下2桁） |
| `{mm}` | 3 | 月（0埋めなし） |
| `{dd}` | 2 | 日（0埋めなし） |
| `{ggge}` | 令和8 | 和暦（1年の場合は「令和元」になります） |
| `{aaa}` | 月 | 曜日（漢字1文字） |
**テンプレートの記述例：**
> `{ggge}年{mm}月{dd}日({aaa}) 定例作業のご報告`
> ↓ (マクロで作成後)
> `令和8年3月2日(月) 定例作業のご報告`
## 必要要件
* Windows 10 / 11
* Microsoft Outlook (Classic Desktop)
* **共通モジュール**: `modLogger.bas` (本リポジトリに含まれる共通ログモジュール)
* **専用ユーザーフォーム**: `frmSelectTemplate.frm` / `frmSelectTemplate.frx`
## インストール
1. このリポジトリのファイルをダウンロードします。
2. Outlookを起動し、`Alt + F11` でVBAエディタを開きます。
3. **モジュールとフォームのインポート**:
    * `ファイル` > `ファイルのインポート` から以下のファイルをインポートします。
        1. `src/modLogger.bas` (共通ログモジュール)
        2. `src/modTemplateMail.bas` (メイン機能モジュール)
        3. `src/frmSelectTemplate.frm` (テンプレート選択用ユーザーフォーム) ※ `.frx` も自動で読み込まれます。
4. **ThisOutlookSessionの設定**:
* VBAエディタ左側の `Project1` > `Microsoft Outlook Objects` > `ThisOutlookSession` をダブルクリックします。
* 以下のプロシージャを追加します（リボンやクイックアクセスツールバーからの呼び出し用です）。
    ```vb
    ' テンプレートから新規メールを作成するマクロ
    Public Sub テンプレートからメールを作成()
        modTemplateMail.CreateMailFromTemplate
    End Sub
    ```
## 設定 (Configuration)
テンプレートを格納するフォルダの場所は、共通設定ファイル `%APPDATA%\OutlookVBA\config.ini` の **`[TemplateMail]`** セクションで設定します。
1. **設定ファイルの準備**:
    * エクスプローラーで `%APPDATA%\OutlookVBA\` を開きます。
    * `config.ini` をテキストエディタで開きます。
2. **設定の編集**:
    * 以下のセクションを追記・編集します。（文字コードは **UTF-8 (BOMなし)**）
    * `%APPDATA%` などの環境変数が使用可能です。
        ```ini
        [TemplateMail]
        # テンプレートファイル（.oft または .msg）を格納するフォルダのパス
        # 指定したフォルダが存在しない場合は、マクロ初回実行時に自動作成されます。
        TemplateFolder=%APPDATA%\OutlookVBA\Templates
        ```
## 使い方
1. **テンプレートの準備**:
    * Outlookで新規メールを開き、件名や本文に変数（`{ggge}` など）を含めた定型文を作成します。
    * `ファイル` > `名前を付けて保存` を選び、ファイルの種類を **Outlook テンプレート (*.oft)** にして、設定したフォルダ（例: `%APPDATA%\OutlookVBA\Templates`）に保存します。
2. **マクロの実行**:
    * Outlookのメイン画面で、マクロ `テンプレートからメールを作成` を実行します。（リボンやクイックアクセスツールバーに登録推奨）
    * 初回実行時、保存先フォルダがない場合は自動作成され、案内メッセージが出ます。
3. **テンプレートの選択**:
    * 保存されているテンプレートの一覧がウィンドウで表示されるので、目的のものを選んで「作成」をクリックします。
4. **メールの表示**:
    * 日付が自動置換され、テキスト形式に変換された状態の新規メールが立ち上がります。
## ログ
本ツールは `modLogger` を使用して動作ログを記録します。
* **ログ保存場所**: `%APPDATA%\OutlookVBA\logs\`
* **ファイル名**: `yyyy-mm-dd.log` (日付ごとのローテーション)
* **ログ出力例:**
    ```text
    2026/03/03 09:30:00.123 [260303-093000-TMPL] [TemplateMail] === START テンプレート作成処理 ===
    2026/03/03 09:30:00.150 [260303-093000-TMPL] [TemplateMail] 選択ダイアログ表示
    2026/03/03 09:30:02.500 [260303-093000-TMPL] [TemplateMail] 選択されたテンプレート: C:\Users\Username\AppData\Roaming\OutlookVBA\Templates\日報テンプレート.oft
    2026/03/03 09:30:02.800 [260303-093000-TMPL] [TemplateMail] 置換・テキスト形式変換完了、メールを表示
    2026/03/03 09:30:02.810 [260303-093000-TMPL] [TemplateMail] === END テンプレート作成処理 ===
    ```
